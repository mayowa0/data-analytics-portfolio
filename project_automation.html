Overview
The Problem Being Solved

At SeedFi, loan performance data needed to reach the leadership team every week and every month — covering both merchant users (businesses taking loans) and individual users. Previously this involved manually querying the database, compiling numbers, and drafting emails.

I automated this entirely. Four Python scripts now run on schedule, each pulling fresh data from the PostgreSQL database, computing all the KPIs, building a styled HTML email body, generating an Excel attachment, and dispatching everything via the ZeptoMail API — with zero human intervention required.

Code Highlight
Core SQL Query Pattern

Each script runs two queries — current period and previous period — then calculates the delta. Credentials are always loaded from environment variables, never hardcoded.

# All credentials loaded from .env — never hardcoded import os from dotenv import load_dotenv load_dotenv() DB_HOST = os.getenv('DB_HOST') # e.g. "db.yourcompany.com" DB_NAME = os.getenv('DB_NAME') DB_USER = os.getenv('DB_USER') DB_PASS = os.getenv('DB_PASS') # Rolling 7-day window calculated automatically today = datetime.now().date() curr_start = today - timedelta(days=7) prev_start = curr_start - timedelta(days=7) prev_end = curr_start # Fetch both periods, build summaries, calculate week-on-week change curr_df = fetch_data(curr_start, today) prev_df = fetch_data(prev_start, prev_end) cs = build_summary(curr_df) ps = build_summary(prev_df) # pct_change helper handles division-by-zero gracefully def pct_change(curr, prev): if prev == 0: return 100.0 if curr > 0 else 0.0 return ((curr - prev) / prev) * 100
